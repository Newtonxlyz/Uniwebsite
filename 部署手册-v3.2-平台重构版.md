# lvyz.org 部署操作手册（v3.2 — Next.js 16 平台重构版）

> 版本：v3.2 | 修订日期：2026-06-16
> 基于 v3.1 静态仓库方案，**重构为单一 Next.js 16 平台 + Better Auth 统一认证**
> 目标：**全 5 子站（crashai / kids-ai / picturebook / kb / blog）+ 主站，统一一套代码，统一一套认证，统一一套数据库**

---

## 〇、v3.2 核心改进（对比 v3.1）

| 改进项 | v3.1 现状 | v3.2 解决方案 |
|---|---|---|
| 代码架构 | 静态 HTML + Vite 子站混合 | **单一 Next.js 16 monorepo** (App Router + RSC) |
| 认证系统 | 各子站独立 / 无认证 | **Better Auth + Prisma + PostgreSQL**（全站统一） |
| 内容管理 | MDX/JSON 文件 | **PostgreSQL 数据库 + CMS 后台** |
| 媒体上传 | 手动放 R2 / Cloudflare | **/api/blog/upload API + 自动 R2 集成** |
| 第三方嵌入 | 无 | **小红书 / B 站 / YouTube 内嵌支持** |
| 部署 | Vercel + 5 个独立静态项目 | **Vercel 单项目 + 1 套代码** |

## 二、本地开发

### 2.1 环境要求

- **Node.js 22+**（Next.js 16 需要）
- **pnpm 10+**
- **Git**

### 2.2 克隆与初始化

```powershell
git clone https://github.com/Newtonxlyz/MinimaxWeb.git D:\LvyzWeb\platform
cd D:\LvyzWeb\platform
pnpm install
```

### 2.3 配置环境变量

```powershell
cp .env.example .env
notepad .env
```

填入：

```env
# 应用
NEXT_PUBLIC_APP_URL="http://localhost:3000"
NODE_ENV="development"

# 数据库（本地用 SQLite，部署到 Vercel 改 postgresql）
DATABASE_URL="file:./dev.db"

# Better Auth
BETTER_AUTH_SECRET="生成：openssl rand -hex 32"
BETTER_AUTH_URL="http://localhost:3000"

# Cloudflare R2（可选，不填上传会失败）
R2_ACCOUNT_ID=""
R2_ACCESS_KEY_ID=""
R2_SECRET_ACCESS_KEY=""
R2_BUCKET_NAME="lvyz-media"
R2_PUBLIC_URL="https://media.lvyz.org"
```

### 2.4 初始化数据库 + Seed

```powershell
pnpm db:generate       # 生成 Prisma Client
pnpm db:push           # 创建表（开发用）
pnpm db:seed           # 种子数据（管理员 + 示例文章）
```

默认管理员：`admin@lvyz.org / lvyz2026`

### 2.5 启动 dev server

```powershell
pnpm dev
# 访问 http://localhost:3000
```

## 四、认证体系（Better Auth）

### 4.1 用户角色

```typescript
type Role = "GUEST" | "USER" | "SUBSCRIBER" | "EDITOR" | "ADMIN" | "SUPERADMIN";
```

| 角色 | 权限 |
|---|---|
| GUEST | 未登录，只读 |
| USER | 评论 + 自己文章 CRUD |
| SUBSCRIBER | 订阅专属内容 |
| EDITOR | 编辑/审核所有内容 |
| ADMIN | 用户管理 + 全部内容 |
| SUPERADMIN | 系统设置 |

### 4.2 客户端用法

```typescript
import { useSession, signIn, signOut } from "@/lib/auth-client";

function MyComponent() {
  const { data: session } = useSession();
  const user = session?.user;

  if (!user) return <Link href="/login">登录</Link>;
  return <button onClick={() => signOut()}>退出 ({user.name})</button>;
}
```

### 4.3 服务端用法

```typescript
import { getCurrentUser, requireUser, requireAdmin } from "@/lib/posts";

// 可选登录
const user = await getCurrentUser();

// 必须登录
try {
  const user = await requireUser();
} catch {
  // 401
}

// 必须管理员
try {
  const user = await requireAdmin();
} catch {
  // 403
}
```

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/blog/posts/[id]/comments` | 列出 |
| POST | `/api/blog/posts/[id]/comments` | 添加（需登录） |
| DELETE | `?commentId=xxx` | 删除（作者或管理员） |

### 5.3 媒体上传

```typescript
// 上传图片/视频/音频
const form = new FormData();
form.append("file", file);  // max 100MB
const res = await fetch("/api/blog/upload", { method: "POST", body: form });
const { url, key } = await res.json();
// 在 Markdown 里引用：![alt](url)
```

### 5.4 嵌入第三方内容

文章 `embeds` 字段是 JSON 数组，支持：

| 平台 | URL 格式 | 渲染方式 |
|---|---|---|
| B 站 | `https://www.bilibili.com/video/BV...` | 官方 player iframe |

## 六、Cloudflare R2 配置

### 6.1 创建 R2 Bucket

1. Cloudflare Dashboard → **R2** → **Create bucket**
2. 名称：`lvyz-media`
3. 区域：Automatic
4. 创建后 → **Settings** → **Public Access** → 启用 `media.lvyz.org` 自定义域

### 6.2 创建 API Token

1. Cloudflare Dashboard → **My Profile** → **API Tokens** → **Create Token**
2. 模板：**Custom token**
3. 权限：`Account:Cloudflare R2:Edit`
4. 区域：Specific zone → `lvyz.org`
5. 复制 `Access Key ID` 和 `Secret Access Key`

### 6.3 填入 .env

```env
R2_ACCOUNT_ID="你的 Cloudflare Account ID"
R2_ACCESS_KEY_ID="上面创建的 Access Key ID"
R2_SECRET_ACCESS_KEY="上面创建的 Secret"
R2_BUCKET_NAME="lvyz-media"
R2_PUBLIC_URL="https://media.lvyz.org"
```

## 七、部署到 Vercel

### 7.1 准备工作

- Vercel 账号（已绑定 GitHub）
- PostgreSQL（推荐 [Prisma Postgres](https://console.prisma.io) 或 Vercel Postgres）
- 域名 `lvyz.org`（Cloudflare 管理）

### 7.2 创建 PostgreSQL

**方案 A：Prisma Postgres（推荐）**

1. 打开 https://console.prisma.io → 注册/登录
2. 创建 **Postgres** 数据库
   - Name: `lvyz-platform`
   - Region: 选 **Asia Pacific (Singapore)** 或 **US East**（最近的 Vercel 区域）
   - Plan: Free tier（1GB 存储 + 充足连接数）
3. 创建后 → **Connection String** → **Prisma ORM** 标签
4. 复制完整 URL，格式：
   ```
   postgres://[user]:[password]@db.prisma.io:5432/postgres?sslmode=require
   ```
5. **同时**会生成两个环境变量名：
   - `POSTGRES_URL`（带连接池）
   - `PRISMA_DATABASE_URL`（直连，给 Prisma Migrate 用）
6. 我们项目用 `DATABASE_URL`（无连接池的直连更稳定）

**方案 B：Vercel Postgres**

1. Vercel Dashboard → **Storage** → **Create Database** → **Postgres**
2. Region: **Washington, D.C. (iad1)**
3. Plan: **Hobby (Free)**
4. 复制 **POSTGRES_URL**（非 pooler 版本）

### 7.3 部署项目

1. GitHub 仓库已 push（`Newtonxlyz/MinimaxWeb`）
2. Vercel Dashboard → **Add New Project** → 选 `MinimaxWeb`
3. **Environment Variables** 填入：

```env
NEXT_PUBLIC_APP_URL = "https://lvyz.org"
DATABASE_URL = "postgres://[user]:[password]@db.prisma.io:5432/postgres?sslmode=require"
BETTER_AUTH_SECRET = "openssl rand -hex 32 生成"
BETTER_AUTH_URL = "https://lvyz.org"
R2_ACCOUNT_ID = "..."
R2_ACCESS_KEY_ID = "..."
R2_SECRET_ACCESS_KEY = "..."
R2_BUCKET_NAME = "lvyz-media"
R2_PUBLIC_URL = "https://media.lvyz.org"
```

4. **Build Command**: `prisma generate && next build`（已在 package.json）
5. **Output Directory**: `.next`
6. 点击 **Deploy**

### 7.4 首次部署后初始化 DB（重要！）

部署完成后，**数据库是空的**，需要：

**方案 A：本地用生产 URL 初始化（推荐）**

```powershell
cd D:\LvyzWeb\platform
# 临时把 DATABASE_URL 切到生产
$env:DATABASE_URL = "postgres://[user]:[password]@db.prisma.io:5432/postgres?sslmode=require"
pnpm db:push
pnpm db:seed
# 然后 unset 避免影响本地开发
Remove-Item Env:DATABASE_URL
```

**方案 B：加个 /api/admin/init 端点**

部署后 GET 一次，触发 `db push + seed`（不安全，只用于首次部署）

### 7.5 域名绑定

Vercel Dashboard → 项目 → **Settings** → **Domains** → 添加 `lvyz.org` 和 `www.lvyz.org`
按提示在 Cloudflare DNS 加 CNAME 记录。

### 7.6 本地开发 vs 部署

| 场景 | DATABASE_URL | schema provider |
|---|---|---|
| 本地开发 | Prisma Postgres（推荐）或 Vercel Postgres | `postgresql` |
| 部署 | 同上 | `postgresql` |

⚠️ **本项目不再支持 SQLite**（schema 已切到 postgresql，因为 Prisma Postgres 免费层够好）。
如要切回 SQLite，改 `prisma/schema.prisma` 的 `provider = "sqlite"`，并把所有 `String?` 默认值检查一遍。

## 九、监控与维护

### 9.1 Vercel 监控

- Dashboard → 项目 → **Analytics** 看流量
- **Functions** 看 API 错误
- **Logs** 看实时日志

### 9.2 数据库备份

Vercel Postgres 自动每日备份，可在 Dashboard → Storage → Postgres → Settings 调整频率。

### 9.3 常见问题

**Q1: /blog/[slug] 404？**
A: slug 必须是 ASCII（不能含中文）。看 `src/lib/posts.ts` 的 `slugify()`。

**Q2: 上传图片失败？**
A: 检查 R2 环境变量。`/api/blog/upload` 会返回具体错误。

**Q3: 登录后跳转到 404？**
A: 检查 `.env` 的 `BETTER_AUTH_URL` 与 `NEXT_PUBLIC_APP_URL` 是否匹配实际域名。

**Q4: 留言不显示？**
A: 检查 post 是否设置了 `allowComments: true`，comment 状态 `VISIBLE`。

**Q5: 部署后 Prisma 报错？**
A: 确认 `DATABASE_URL` 在 Vercel 是 postgresql 格式（含 sslmode=require）。
